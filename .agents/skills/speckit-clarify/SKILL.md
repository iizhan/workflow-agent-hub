# Spec Kit Clarify

Reduce ambiguity before planning.

## Workflow

1. Read the requirement analysis or `spec.md`.
2. Ask only questions that materially affect scope, permissions, data behavior, external dependencies, or user experience.
3. Prefer at most 3 high-value clarification points.
4. After clarification, update the relevant artifact instead of leaving ambiguity only in chat history.
