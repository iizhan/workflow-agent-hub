# 品牌与界面结构设计

## 1. 本轮目的

在目标、边界、领域模型和信息架构已经明确后，这一轮回答：

- 新产品应该呈现出什么品牌气质
- 首页应该先表达什么
- 应用列表和应用详情应该如何建立“工作台感”
- 新界面如何和旧后台式体验拉开差异

本轮仍然不进入具体实现代码，只建立视觉与页面结构方向。

## 2. 品牌设计目标

新产品不再延续旧的 `Hermes Web UI` 心智，因此界面必须帮助用户完成一次认知迁移：

- 从“系统控制台”迁移到“项目执行工作台”
- 从“聊天工具集合”迁移到“应用工作空间”
- 从“底层技术能力入口”迁移到“围绕目标工作的入口”

## 3. 品牌人格

建议新品牌人格定义为：

- 冷静
- 清晰
- 稳定
- 有判断力
- 面向长期工作

不建议的人格方向：

- 玩具化
- 过度赛博
- 过度黑客终端风
- 过度营销化
- 传统企业后台风

## 4. 视觉气质结论

### 推荐方向

推荐采用：

- light-first workbench
- editorial typography
- restrained accent
- structured panels

### 不推荐方向

- 全黑 SaaS landing 风
- 纯黑白极简后台风
- 多彩卡片式生产力应用拼贴风
- 过强终端 / IDE 模仿风

## 5. 设计系统基线

本轮已同步建立项目级视觉源文件：

- [DESIGN.md](/Users/bing/MySelf/AI/Hermes/hermes-web-ui/DESIGN.md:1)

这意味着后续所有 UI 设计都应围绕以下基线展开：

- 轻暖中性色大底
- 深墨色正文
- 单一克制强调色
- Serif 只用于身份和标题瞬间
- Sans 承担主要界面
- 用布局和类型建立品牌，而不是用炫技装饰

## 6. 首页结构设计

首页不应默认把用户扔进聊天或系统页。

### 首页的核心任务

首页要回答三个问题：

1. 我现在有哪些应用
2. 我最近在推进什么
3. 我下一步最值得点哪里

### 首页推荐结构

#### 第一屏：工作入口

- 页面标题与一句清晰定位语
- “创建应用”主按钮
- 最近应用卡片
- 最近运行摘要

#### 第二屏：最近成果

- 最近产物列表
- 最近活跃项目
- 最近失败或待处理运行

#### 第三屏：模板与起步

- 应用模板
- 角色模板
- 工作流模板
- 从代码 / 文档 / PPT / 研究开始的入口

### 首页应避免

- 不要第一屏就是系统配置统计
- 不要第一屏是聊天窗口
- 不要第一屏堆满次要监控卡片

## 7. 应用列表页设计

应用列表页是新产品最关键的目录页。

### 它要表达什么

- 这是一个“工作集合页”，不是简单卡片墙
- 应用是长期工作的单位
- 每个应用都应展示可判断的状态，而不是只有名字

### 推荐信息密度

每个应用卡片或列表项应至少展示：

- 应用名称
- 一句话目标
- 场景类型
- 主项目
- 参与角色数
- 最近运行时间
- 最近产物数或状态

### 推荐视觉表达

- 更像项目板块，而不是社交卡片
- 主状态与次状态分层清晰
- 有明确的“进入应用”动作

## 8. 应用详情页设计

应用详情页应成为最核心的“工作台页面”。

### 它的第一原则

用户进入应用详情后，不应该再问：

- 我现在在哪
- 这个应用在做什么
- 下一步该点哪里

### 推荐结构

#### 头部

- 应用名称
- 一句话目标
- 场景标签
- 主操作按钮：
  - 发起运行
  - 查看工作流
  - 查看产物

#### 左侧或二级导航

- Overview
- Projects
- Agents
- Workflow
- Artifacts
- Runs
- Collaboration
- Settings

#### 主内容区

根据当前分区展示一个明确的决策区域，不要多个复杂模块同时抢焦点。

## 9. Overview 页面设计

Overview 是应用详情的第一屏。

### 它应该像什么

它更像“项目驾驶舱”，不是后台报表页。

### 推荐模块

- 应用摘要
- 主项目状态
- 当前活跃角色
- 当前工作流状态
- 最近运行
- 最近产物
- 下一步推荐动作

### 它必须解决

- 用户一眼知道这个应用当前处于什么状态
- 用户一眼知道下一步该做什么

## 10. Collaboration 页面设计

这是旧群聊能力的新落点。

### 它的定位

不是首页，不是顶层主入口，而是应用内部的一种协作视图。

### 它应该保留什么

- 实时对话
- 多角色协作
- 过程消息
- 流程推进中的互动

### 它不再承担什么

- 不再承担项目绑定总入口
- 不再承担工作流总配置入口
- 不再承担产品主线入口

## 11. System 页面设计原则

系统页仍然重要，但视觉上应比应用页更克制、更工具化。

### 区分方式

- 应用页强调任务与产物
- 系统页强调配置与状态

### 视觉差异

- 应用页可以有更强的层级和叙事节奏
- 系统页应更紧凑、更功能型

这样用户会自然感知：

- “这里是做工作”
- “那里是配系统”

## 12. 与旧风格的关键差异

旧风格的问题是：

- 更像工具后台
- 单色系统感过强
- 产品身份主要靠文字，而不是结构和视觉
- 业务主线不够突出

新风格应做到：

- 更强的工作台结构感
- 更清晰的业务 / 系统分层
- 更有辨识度的标题和内容节奏
- 更明确的应用中心视觉

## 13. 本轮结论

品牌与界面结构层最重要的结论是：

- 新产品的视觉不是简单换皮，而是为“应用工作台”身份服务
- 首页要先展示工作入口，不要先展示系统能力
- 应用列表页和应用详情页将成为最重要的品牌承载页面
- 群聊能力要被收编进应用内协作视图
- 系统页应退居管理层，并与应用页形成明显气质差异
